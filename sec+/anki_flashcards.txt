OSI Application Layer; Layer 7. The Application layer is the topmost layer of the OSI model and is responsible for providing network services directly to user applications. It facilitates communication and interaction between software applications and network services. Examples include HTTP, FTP, SMTP, and DNS.
OSI Presentation Layer; Layer 6. The Presentation layer is responsible for data translation, encryption, and formatting to ensure that data exchanged between applications is in a format that the receiving application can understand. It handles tasks such as data compression, encryption, and character encoding.
OSI Session Layer; Layer 5. The Session layer establishes, manages, and terminates communication sessions between devices. It handles session establishment, maintenance, and synchronization, allowing applications on different devices to establish connections and exchange data.
OSI Transport Layer; Layer 4. The Transport layer ensures reliable end-to-end communication between devices. It segments and reassembles data into smaller packets, adds sequence numbers for tracking, and provides error detection and recovery mechanisms. Common protocols include TCP (Transmission Control Protocol) and UDP (User Datagram Protocol).
OSI Network Layer; Layer 3. The Network layer is responsible for logical addressing, routing, and packet forwarding across multiple networks. It uses IP (Internet Protocol) addresses to route packets between source and destination devices. Examples of protocols include IP, ICMP (Internet Control Message Protocol), and routing protocols like OSPF and BGP.
OSI Data Link Layer; Layer 2. The Data Link layer provides error-free transmission of data frames between devices on the same local network segment. It handles framing, addressing, and error detection, ensuring reliable point-to-point and point-to-multipoint communication. Ethernet and Wi-Fi are examples of Data Link layer protocols.
OSI Physical Layer; Layer 1. The Physical layer deals with the physical transmission of data over the network media. It defines the electrical, mechanical, and functional specifications for connecting devices to the network, including cables, connectors, and network interface hardware. Examples include Ethernet cables, optical fibers, and wireless radio frequencies.
TCP/IP Application Layer; Layer 4. Responsible for providing network services directly to user applications. Protocols; HTTP, FTP, SMTP, DNS, Telnet. Corresponds to Layers 5, 6, and 7 of the OSI model (Session, Presentation, and Application layers).
TCP/IP Transport Layer; Layer 3. Manages end-to-end communication between applications on different devices. Provides reliable data delivery, flow control, and error recovery. Protocols; TCP, UDP. Corresponds to Layer 4 (Transport layer) of the OSI model.
TCP/IP Internet Layer; Layer 2. Handles addressing, routing, and packet forwarding between devices across different networks. Uses IP for logical addressing. Protocols; IP (IPv4 and IPv6), ICMP. Corresponds to Layer 3 (Network layer) of the OSI model.
TCP/IP Link (Network Access) Layer; Layer 1. Deals with the physical transmission of data within a single network segment (LAN). Defines protocols for accessing the physical network media, addressing devices, and error detection. Protocols; Ethernet, Wi-Fi (802.11), PPP (Point-to-Point Protocol). Corresponds to Layers 1 and 2 (Physical and Data Link layers) of the OSI model.
